## Golden rules
- **Never read or modify any `.env` file.** Secrets live there; `.env.example` lists the variables.
- Commit to `main`; end commit messages with `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **Railway deploys the backend only on green CI** (5 jobs: frontend, backend, security, a11y, responsive). **Vercel prod web does NOT auto-deploy** тАФ ship it manually: `vercel --prod` then `vercel alias set frontend-<hash>-investikid.vercel.app app.investikid.ai` (the custom domain is pinned and won't auto-follow). A git push alone ships nothing to web. See lines above + `docs/deployment-environments.md`.
- TDD; minimal focused changes. DB changes = hand-written chained Alembic migration (check `alembic heads` first).
- Async backend tests use `pytestmark = pytest.mark.asyncio(loop_scope="session")` + the `client`/`admin_client`/`db_session` fixtures тАФ never a raw `AsyncClient` on the app engine.
- LLM output is always moderated (`moderate_output`); premium-gate with `is_premium`; rate-limit LLM endpoints. It's a kids' app тАФ keep it safe and WCAG 2.2 AA accessible.
- iOS: keep form controls тЙе16px on touch (no `maximum-scale`); keep the WKWebView re-layout + safe-area handling.
- **After every push to `main`, update the progress docs as part of the same task** тАФ move the shipped item into "Live in prod" in `docs/MASTER-BACKLOG.md` and refresh `docs/superpowers/PROGRESS.md` / the roadmap entry. A change isn't done until the trackers reflect it.

## Feature workflow
For any non-trivial feature/change: **design spec тЖТ implementation plan тЖТ TDD**. Write a short spec to `docs/superpowers/specs/YYYY-MM-DD-<topic>-design.md` (goal, scope, data/API, UX, edge cases, test plan), then a task-by-task plan in `docs/superpowers/plans/`, then implement test-first (failing test тЖТ minimal code тЖТ commit per task). Verify (ruff + pytest; tsc + lint + test + build, plus `vitest-axe` for new UI), push, confirm green CI. Surface trade-offs instead of assuming; minimum code; touch only what the task needs.
- **UI/visual enhancements go through Figma first.** For any non-trivial UI or visual change (new screens, redesigns, illustrations, avatar/cosmetics/collectables art, theming), mock up the direction in **Figma** (via the Figma MCP), get it approved, then translate the approved direction to code тАФ so the result looks professional and consistent, not improvised. Trivial tweaks (spacing, copy, reusing an existing pattern) are exempt.

## Domain invariants & criteria (honor in future changes)
These are settled decisions тАФ match them unless the user explicitly changes them.

- **Web auth is same-site.** Prod web (`app.investikid.ai`) calls the API at **`https://api.investikid.ai`** (a Railway custom domain on the prod backend), never `investikid.up.railway.app`. They share `investikid.ai`, so the `SameSite=None` auth cookie is first-party; the railway host is cross-site and Safari/Chrome block the cookie тЖТ login shows "Could not start your session". The Vercel **Production** `VITE_API_BASE_URL` must stay `https://api.investikid.ai`; verify from the live bundle (`grep -c api.investikid.ai` =1, railway =0). Spec: `docs/superpowers/specs/2026-06-15-web-auth-same-site-api-design.md`.
- **Content authoring (market content generation).** Write in the market's English variant тАФ **American English for US/CA** (`mom/neighbor/color/check/soccer`, never `mum/neighbour/colour/cheque/maths/paper round`), British elsewhere (`_market_english`). Ground every example in the verified `MarketBrief` (local products/regulators/currency) тАФ never UK-specific products in non-GB markets. Keep lessons **diverse**: rotate character names/genders/backgrounds + scenarios, blend card/quiz/scenario formats, and use the sibling-`avoid` list so a level's questions aren't near-duplicates. **Spread concepts** across distinct facets per level тАФ don't stack 3+ concepts on one product/account (a market's dominant product, e.g. ISA/MPF/FSCS, may anchor its *own dedicated* level, but mixed with other angles). Tiered depth = **10 / 15 / 20** lessons for complexity tier 1/2/3.
- **Simulator apply-missions.** Mission *completion* (`evaluate_apply_missions`) reads **global** portfolio state, so it MUST be scoped to the child's `active_market_code` тАФ otherwise one action completes every market's same-type mission and multiplies XP+cash. Keep missions **sparse/curated** (never bulk-create the same mission_type across many lessons/markets). Investing modules auto-attach **one** mission to their culminating lesson (`investing_missions.sync_investing_missions`): growth/compoundтЖТ`first_buy`, risk/diversificationтЖТ`diversify`(n=3); 20 XP + admin-editable per-market cash (`simulator.investing_mission_cash`). `publish_market` re-syncs after a republish; backfill via `sync-investing-missions.yml`.
- **Beta delivery + prod safety** (also above): straight-to-`main` in beta; **ask before any prod DB migration** whether to snapshot; manual Vercel prod deploy = `vercel deploy --prod --archive=tgz --yes` then `vercel alias set <slug> app.investikid.ai` (the `--prod` deploy does NOT auto-switch the alias); new `/internal/*` cron POSTs need a CSRF allowlist entry.
