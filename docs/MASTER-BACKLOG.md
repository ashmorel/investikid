- **MoneyWord daily-reset fix (Home card stuck on "Done") — ✅ shipped to prod 2026-06-25** (main `6efd476`, CI green, **frontend-only, no migration**; web aliased to `frontend-jyqrupdia`; `cap sync ios`). The daily MoneyWord resets at **00:00 UTC** (backend keys `ArcadeDailyPlay` rows on `datetime.now(UTC).date()`; a new UTC day returns a fresh board — server was always correct). The bug was client cache: `useMoneyWordToday` used a **static** key `['arcade','moneyword','today']` (no date), so with `gcTime:24h` + no `staleTime` React Query kept serving yesterday's `completed:true` under the same key after midnight — the cached Home `ArcadeDailyCard` label showed "Done" into the next day (the game page fetches live on mount, so the puzzle itself had reset; only the label was stale). Fix: encode the UTC day in the key (`[...,'today', <YYYY-MM-DD>]`) so the rollover changes the key → fresh refetch. Regression test asserts the key carries today's UTC date. tsc/lint clean; new api test + 6 MoneyWord + 2 Arcade tests pass. **Gotcha for future daily features:** any *cached* `useQuery` for daily/periodic-reset content must encode the period in its key.
